# discovery: Provider から UI 型依存を分離（Port/Adapter 再整理）

ステータス: [consumed]
担当: @tkg-engineer
開始: 2025-09-30 00:00:00 +0900
優先度: P0
提案元: legacy-backlog
関連ストーリー: 20250930-provider-ui-decouple

## 目的
- `internal/game/data.TableProvider` を純参照Portにし、UI 型依存を排除して移植性/テスト容易性を高める。\n
## 背景
- 現行 `UserUnitByID(id) (uicore.Unit, bool)` が UI 型を返し、Provider が UI に結合している。\n
## DoD候補
- Provider から `UserUnitByID` を撤去し、`service/ui/adapter` に `User→Unit` 変換を移管。呼び出し箇所を置換し `make mcp` グリーン。\n
## 関連
- internal/game/data/provider.go, internal/game/service/ui/adapter, docs/ARCHITECTURE.md\n
