## 目的
- `docs/` 配下の役割を整理し、システム仕様とゲーム仕様が見通しよく参照できるディレクトリ構成にする。

## 背景
- 現状、`docs/architecture/README.md` や `docs/SPECS/reference/api.md` などシステム仕様系ドキュメントと、将来追加したいゲーム仕様系ドキュメントの置き場が同じ階層に混在している。
- システム仕様は専用ディレクトリへ整理し、別途ゲーム仕様用のディレクトリを用意しておくことで、後続のドキュメント追加・参照をしやすくしたい。

## DoD候補
- [ ] システム仕様向けドキュメント（例: ARCHITECTURE, API）がディレクトリ化され、役割が明確になっている。
- [ ] ゲーム仕様向けの新しいディレクトリ（例: `docs/game/`）が追加されている（中身は空、もしくは README 程度で可）。
- [ ] 既存 README などからの参照方針（どこに何を書くか）が大まかに合意されている。
