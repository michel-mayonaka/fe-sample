# 02_作業カテゴリ/プレフィックス体系の設計

ステータス: [未着手]
担当: @tkg-engineer
開始: 2025-11-19 02:19:57 +0900

## 目的
- 棚卸しで抽出した作業パターンをもとに、ストーリー用の作業カテゴリ集合を決定する。
- 新規ストーリーディレクトリ名のスラグ先頭に付与するプレフィックス（例: `spec-`/`research-`/`impl-`/`docs-`/`workflow-`）のルールを設計する。

## 完了条件（DoD）
- [ ] 採用する作業カテゴリの一覧が決まり、それぞれの対象範囲・代表例・除外例が整理されている。
- [ ] ディレクトリ名フォーマット（`stories/YYYYMMDD-<category>-<slug>/`）と `<category>` に使用できるプレフィックスの定義が文章として確定している。
- [ ] 既存ストーリーへの適用方針（新規のみ必須/過去分を段階的にリネームするなど）の案が整理されている。

## 作業手順（概略）
- 01 の棚卸し結果をもとに、似通ったパターンを統合しつつ、実運用で扱いやすいカテゴリ数に収束させる。
- それぞれのカテゴリについて、想定スコープ・代表 DoD 観点・prefix 名の候補を比較検討し、命名を決める。
- 既存ストーリー（特に `stories/finish`）の prefix 移行をどのように行うか（やる/やらない/別ストーリーに切り出す）を検討し、方針をメモする。
