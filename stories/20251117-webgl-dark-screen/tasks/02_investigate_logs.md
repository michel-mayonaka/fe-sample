# 02_investigate_logs — ログ調査と原因候補の洗い出し

ステータス: [完了]
担当: @kyosuke_tsubo
開始: 2025-11-17 01:13:08 +0900

## 目的
- ブラウザコンソールや Network/Canvas 関連のログを調査し、暗転の直接原因となりうる事象を列挙する。

## 完了条件（DoD）
- [x] コンソールログに出ているエラー/警告が洗い出され、要約されている。
- [x] ネットワークタブから、アセットロード失敗や 404/403 などがないか確認されている。
- [x] WebGL/Canvas 関連のエラー（コンテキストロストなど）がある場合、その内容がメモされている。

## 作業手順（概略）
- 対象 WebGL ビルドをブラウザで開き、Developer Tools（Console/Network）を確認する。
- 必要に応じてローカルビルドと比較し、差分があるログを整理する。
- 原因候補を箇条書きで整理し、対策の方向性を Story 本文または次タスクへ引き継ぐ。

## 進捗ログ
- 2025-11-17 01:13:08 +0900: タスク作成。
- 2025-11-17 01:21:00 +0900: ブラウザコンソールログに `panic: runtime error: invalid memory address or nil pointer dereference`（スタックトレース上は `JSONUserRepo.Table(0x0)`）が出ていることを確認。ユーザRepo初期化失敗時に nil ポインタのまま Table() を呼んでいたことが原因と判断。
