# 04 生成物ディレクトリ統一と無視設定

## 目的
ビルド/テスト生成物の出力先を統一し、クリーンと無視設定を明確にする。

## 方針（ドラフト）
- 生成物出力: `out/` へ集約（WASM は `site/` でも可、要整理）。
- 無視設定: `.gitignore` へ追加提案（実装は開始指示後）。
- クリーン: `make clean` で出力物を完全削除。

## 成功条件（DoD）
- 出力先の命名/階層/掃除手順が合意される。

## 実装メモ（結果）
- 出力先: `out/` に統一（`out/bin`, `out/logs`, `out/coverage`）。
- クリーン: `make clean` を追加。
- 無視設定: `.gitignore` に `out/` と `site/` を追加（WASM等の将来拡張に対応）。
- 整理: `.gitignore` の `site/` 重複エントリを削除（2025-11-16）。
