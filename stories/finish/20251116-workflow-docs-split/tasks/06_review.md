# 06_review — 検証と最終レビュー

## 目的
- WORKFLOW ドキュメント再構成後の整合性と可読性を確認し、ストーリーの DoD を満たしていることを検証する。

## 作業項目
- 新しい構成を前提に、ストーリー作成〜実装〜検証の流れを辿れるかをチェックする。
- CI 方針やローカル開発手順が適切なファイルから辿れるかを確認する。
- `make mcp` を実行し、ビルド/テスト/ドキュメント整合に問題がないかを確認する。

## アウトプット
- レビュー結果のメモ（気づき/今後の改善候補を含む）。
- DoD を満たしたことの確認（README のチェックボックス更新など）。
