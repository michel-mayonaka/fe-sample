# 002 ドメイン I/F とイベント定義

## 目的
- `pkg/game/input` に入力イベント/状態/I/F を追加し、UI 依存のない取得手段を提供する。

## スコープ
- `InputEvent`/`ControlState`/`InputSource` の定義。
- マッピング用テーブル（デフォルトレイアウト）雛形。
- 最小のユニットテスト（ゼロ値/状態遷移）。

## 成果物
- 追加コード: `pkg/game/input/...`（新規パッケージ）。
- テスト: `pkg/game/input/..._test.go`。

## 受け入れ基準
- [x] `pkg/game` のビルドとテストが通る。
- [x] 外部パッケージへの UI 依存 import がゼロ。
- [x] `ControlState` の更新と等価性判定がテストで確認できる。

## 実装ヒント
- `ControlState` は bool/float32 の組み合わせ（押下/アナログ軸）。
- 連打/リピートは上位で扱えるよう、押下/トグル/長押し時間のカウンタも検討。
