## 目的
- 現在のコミットメッセージおよび PR テンプレ/本文の運用実態を把握し、Story 参照や DoD/検証手順の記載状況を整理する。

## 完了条件（DoD）
- [ ] 直近の PR/コミットいくつかをサンプルとして確認し、Story 記載の有無や書き方のパターンがメモされている。
- [ ] 現行フローで問題になっている点（例: ストーリー特定に時間がかかる状況）が具体例付きで列挙されている。
- [ ] どの程度の粒度で Story 記載を必須化するか（例: すべての PR / 一定規模以上のみ）の叩き台が用意されている。

## 作業手順（概略）
- git log / GitHub の PR 一覧から直近のいくつかをピックアップする。
- Story 記載の有無・書式・DoD/検証手順の扱いを確認し、代表パターンをまとめる。
- 問題になっているケースと、良い例になっているケースを比較し、要件のインプットとする。
