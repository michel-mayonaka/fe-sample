# 02_PRテンプレ案と運用ルールのドラフト

ステータス: [未着手]
担当: @tkg-engineer
開始: 2025-11-17 02:02:11 +0900

## 目的
- Story/DoD/検証手順を自然に書ける `.github/PULL_REQUEST_TEMPLATE.md` の案を作成し、運用ルール（必須/任意など）を整理する。

## 完了条件（DoD）
- [ ] Story ID・概要・DoD・検証手順を記載する欄を含んだ PR テンプレ案が用意されている。
- [ ] 軽微な変更やドキュメント修正などに対する扱い（Story をどう書くか/書かなくてよいか）のポリシーがドラフトとしてまとめられている。
- [ ] テンプレ案と運用ルールが、後続の docs 反映タスクでそのまま利用できる粒度になっている。

## 作業手順（概略）
- 既存の PR テンプレ（あれば）や、他プロジェクトの事例を参考にセクション構成を検討する。
- `Story: YYYYMMDD-slug` のような書式や、複数ストーリーにまたがる場合の扱いを決める。
- テンプレ本文と運用ルールのメモを作成し、レビューしやすい形にまとめる。
