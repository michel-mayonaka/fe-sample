## 目的
- 実際に `pkg/game` がどこからどのように import されているかを棚卸しし、可視性変更時の影響範囲を明確にする。

## 完了条件（DoD）
- [ ] `pkg/game` を import しているパッケージ一覧と用途（例: 戦闘解決/入力定義など）が整理されている。
- [ ] 外部モジュール（本レポジトリ外）からの利用有無が明示されている。
- [ ] 可視性を `internal` に変更した場合にビルドエラーになる箇所の概算が分かるメモがある。

## 作業手順（概略）
- `rg "ui_sample/pkg/game" -n` などで import 実態を列挙する。
- 利用している型/関数の粒度をざっくり分類し、「ドメインコア」「補助的な構造体」などのカテゴリに分ける。
- 可視性変更時の影響（ビルドエラー箇所/テスト）をテキストベースで見積もる。
