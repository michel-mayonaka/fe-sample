# gameplay 仕様テンプレート（ユースケース/ロジック）

> このファイルをコピーして、戦闘/在庫/ユースケースなどロジック中心の仕様を記述します。

## 1. 概要
- 機能名 / ユースケース名
- 目的（どのような価値を提供するか）

## 2. 対象範囲
- この仕様がカバーする範囲
- 他仕様（画面仕様/ドメイン仕様など）との関係

## 3. 用語・前提
- 用語定義
- 前提条件 / 外部システム / 設定値 など

## 4. ユースケース / シナリオ
- 代表的なシナリオを箇条書きまたはテーブルで整理
- 異常系 / 例外系があれば合わせて記述

## 5. 振る舞い（ロジック）
- 入力 → 出力 の関係
- 状態遷移や計算式があれば明示
- エッジケースの扱い

## 6. 永続化 / データ
- 関連するテーブル/JSON/構造体
- 読み書きのタイミングと整合性のルール

## 7. 非機能 / 制約
- パフォーマンス/リソース/制限事項 など

## 8. テスト観点
- 単体テスト・結合テストで最低限確認したい観点

## 9. 備考
- メモ/TODO/今後の検討事項
