# 画面仕様テンプレート（ui spec）

> このファイルをコピーして、画面単位の仕様（ステータス画面/在庫画面など）を記述します。

## 1. 画面概要
- 画面名
- 目的（ユーザは何を達成できるか）
- 対象ユーザ（例: プレイヤー/開発者向けデバッグ など）

## 2. レイアウト
- 主なコンポーネント（パネル/リスト/ボタンなど）の配置
- 重要な座標やサイズがあれば簡易に記述（詳細はコード側 `layout` を参照でも可）

## 3. 表示内容
- 各コンポーネントに何を表示するか（項目名/フォーマット/例）
- 非表示条件や強調表示のルールがあれば記述

## 4. 入力と挙動
- キー入力・マウス操作と、それに対する画面の反応
- フォーカス移動・選択・決定・キャンセルの挙動

## 5. 状態遷移
- この画面へ遷移する条件
- この画面から遷移しうる画面と条件

## 6. エラー・例外
- エラー時の表示内容や遷移
- 入力不正時のフィードバック

## 7. テスト観点
- レイアウト崩れや表示内容確認のチェックリスト
- 入力と挙動の組み合わせテスト観点
