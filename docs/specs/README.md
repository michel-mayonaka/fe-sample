# specs ディレクトリ（仕様ハブ）

`docs/specs/` は、本プロジェクトの「実装に直結する仕様（Specification）」を集約するためのディレクトリです。人間の開発者とコーディングエージェントの両方が同じ仕様を参照し、仕様駆動で開発・レビューできることを目的とします。

## 目的
- システム仕様・画面仕様など、実装とテストの根拠となる情報を1か所に集約する。
- 仕様本文は人間/エージェント共通の1セットとし、二重管理を避ける。
- エージェント向けの「読み方ガイド」「参照優先度」などは別ドキュメントで補完する。

## ディレクトリ構成（案）
- `docs/specs/system/`
  - システム全体の振る舞い・ユースケース・データフローなどの仕様を置く。
  - 例: 永続化の振る舞い、戦闘ルール、在庫操作のシナリオ など。
- `docs/specs/ui/`
  - 画面単位の仕様（画面一覧・レイアウト・状態遷移・入力と反応など）を置く。
  - 例: ステータス画面・在庫画面・戦闘画面の仕様。
- （将来検討）
  - `docs/specs/domain/`: ドメインモデル/集約ルートの仕様
  - `docs/specs/api/`: 外部API連携が増えた場合のAPI仕様

## 仕様ドキュメントの位置づけ
- アーキテクチャの全体像/思想: `docs/ARCHITECTURE.md`
- 命名規約・スタイル: `docs/NAMING.md`, `docs/COMMENT_STYLE.md`
- 実装に直結する振る舞い・画面仕様: `docs/specs/` 配下（本ディレクトリ）

実装やリファクタリングの際は、まず `docs/specs/` で該当する仕様があるか確認し、不足していれば先に仕様を追加/更新する運用を想定しています。

詳細な「読み方」やエージェント向けの注意点は `docs/specs/AGENTS.md` を参照してください。
