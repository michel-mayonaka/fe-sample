# コメント記法ガイドライン（GoDoc 準拠）

本プロジェクトにおけるコメント（コード内）の最小ルールを定めます。詳細仕様や長文は docs/*.md（ARCHITECTURE.md / API.md など）に集約し、コード内は 1–2 行の要約に留めます。

- パッケージコメント: 各パッケージに 1 つ以上、先頭ファイルに役割を 1 行で記述。
- 公開識別子（型/関数/定数/変数）: 「X は …」の形で 1 行要約（日本語）。
- 詳細は docs へ: 実装理由・背景・長文の Note は docs/*.md に記載し、コード側から簡潔に参照を促す。
- 用語: 日本語を基本。英語が自然な一般名詞（Transaction, Provider など）は併記可。
- JSDoc 風タグ（@param/@return）は使用しない（GoDoc の慣習に従う）。
- 例外: 境界・副作用・集約ポリシーなど注意事項は 2–3 行まで補足可。

補足: lint（revive）の `package-comments`/`exported` を満たす目的でも、このガイドに従います。
